Git merge instructions

When you need to synchronise your git clone repository with the official one, please follow the instructions below. The following steps are referred to git version 1.7.1, currently (2016-01-20) in use on lxplus.cern.ch. More infos can be found on the git web site.

Prerequisites

  1. create an account on github.com;
  2. add an ssh public key - instructions can be found here;
  3. fork on github.com. This operation will create your own copy of the main project repo;
  4. change clone URL on your fork on github.com from HTTPS to SSH and then git clone on your local machine using the SSH url (required in order to git push);

Nota Bene: another option to change the clone URL is through the the git configuration file of the clone: 

SixTrack/.git/config
from: 
url = https://github.com/goofy/SixTrack.git
into: 
url = git@github.com:goofy/SixTrack.git

Steps for merging from upstream

  1. cd to your local copy: 
     > pwd
 /afs/cern.ch/user/g/goofy/SixTrack
  2. add a remote “upstream” (first time only): 
     > git remote add upstream git@github.com:SixTrack/SixTrack.git
  3. check out your master branch: 
     > git checkout master
  4. fetch the upstream repo 
     > git fetch upstream
 From github.com:SixTrack/SixTrack
 * [new branch]      SVN-trunk  -> upstream/SVN-trunk
 ...
  5. fast-forward merge upstream’s master into your master: 
     > git merge upstream/master
  6. push merge to github.com 
     > git push
 Counting objects: ...
 To git@github.com:goofy/SixTrack.git
 xxxxxxx..yyyyyyy  master -> master
